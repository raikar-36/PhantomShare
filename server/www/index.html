Чому SecureShare?

Створено для тих, хто цінує приватність та простоту

Наскрізне шифрування

X25519 обмін ключами + AES-256-GCM. Сервер бачить лише шифротекст — ніколи ваші дані.

Без реєстрації

Не потрібен акаунт, email чи телефон. Просто відкрийте програму та поділіться кодом сесії.

До 5 ГБ

Передавайте великі файли без обмежень хмарних сервісів. Відновлення перерваних передач включено.

Авто-перепідключення

Обірвався зв'язок? Програма автоматично перепідключиться та продовжить з місця зупинки.

Верифікація

Код верифікації гарантує, що ви спілкуєтесь саме з потрібною людиною. Захист від MITM-атак.

Без хмарного зберігання

Файли передаються напряму. Нічого не зберігається на сервері — нуль слідів після передачі.

Як це працює?

Три прості кроки — і файл у безпеці

1

Відправник створює сесію

Натисніть «Надіслати», оберіть файл. Програма генерує унікальний код сесії з 8 символів.

⟶
2

Отримувач підключається

Введіть код у себе в програмі. Обидва пристрої обмінюються ключами та верифікують з'єднання.

⟶
3

Файл передається

Дані шифруються на пристрої відправника та розшифровуються тільки у отримувача. Сервер бачить лише шум.

Безпека на першому місці

Кожен аспект розроблений з акцентом на приватність

X25519 + HKDF-SHA256

Еліптичний обмін Діффі-Хеллмана для генерації спільного ключа без передачі секретів по мережі.

AES-256-GCM з AAD

Шифрування з аутентифікацією. Код сесії прив'язаний як Associated Data — запобігає підміні шифротексту.

SHA-256 контроль цілісності

Кожен переданий файл перевіряється хешем. Навіть один змінений байт буде виявлено.

TLS 1.3 транспорт

WebSocket через WSS з автоматичним сертифікатом Let's Encrypt. Подвійний шар шифрування.

Zero-knowledge сервер

Relay сервер ніколи не бачить вміст файлів. Коди сесій хешуються — жодних логів контенту.

Відкритий код

Весь код доступний на GitHub. Перевіряйте, аудитуйте, пропонуйте покращення.
